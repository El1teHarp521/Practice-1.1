* **Frontend:** HTML5, CSS3, Vanilla JavaScript, Service Workers, Web App Manifest.
* **Backend:** Node.js, Express, Socket.IO, Web-Push.

## 🌟 Реализованный функционал

### 1. Безопасность и Авторизация (Практики 7, 8, 9)
*   **Хеширование:** Пароли пользователей никогда не хранятся в открытом виде (используется `bcrypt`).
*   **JWT Auth:** Реализована система двойных токенов. `accessToken` (короткий) для запросов и `refreshToken` (длинный) для безопасного продления сессии.
*   **Автоматизация:** На фронтенде настроены Axios Interceptors. Если срок действия доступа истек, приложение само незаметно для пользователя обновляет токен и повторяет запрос.

### 2. Управление доступом — RBAC (Практика 11)
В системе реализованы 4 роли с разными уровнями доступа:
*   👤 **Гость:** Может только просматривать витрину товаров.
*   🛍 **Пользователь:** Видит товары и детальную информацию (нужен логин).
*   📦 **Продавец:** Может добавлять новые товары и редактировать существующие.
*   ⚡ **Администратор:** Полный доступ. Управление товарами (включая удаление) и управление пользователями (смена ролей, блокировка).

### 3. Интерфейс (Практика 10)
*   **Динамический UI:** Кнопки управления появляются в интерфейсе только если у текущего пользователя есть на это права.
*   **Админ-панель:** Отдельное окно для управления списком пользователей и их ролями.

### 4. Офлайн-режим и PWA (Практики 13, 14, 15)
* **Установка на устройство:** Благодаря настроенному `manifest.json` и набору иконок, приложение можно установить на рабочий стол смартфона или ПК прямо из браузера (кнопка "Install").
* **App Shell & Service Worker:** Приложение кэширует свой каркас (HTML, CSS, JS, шрифты) и загружается мгновенно даже при полном отсутствии интернета.

### 5. Realtime синхронизация (Практика 16)
* **WebSockets (Socket.IO):** При добавлении или удалении задачи в одной вкладке браузера, изменения мгновенно отображаются у всех остальных подключенных клиентов без перезагрузки страницы.
* **Всплывающие In-App уведомления:** Toast-уведомления появляются прямо внутри приложения при активности других пользователей.

### 6. Отложенные Push-напоминания (Практика 17)
* **Таймеры на сервере:** При создании задачи можно указать дату и время напоминания.
* **Системные уведомления:** В назначенное время сервер рассылает payload всем кто включил уведомления , и операционная система (Windows/macOS/Android) показывает нативное Push-уведомление.
* **Интерактивные действия (Snooze):** В самом Push-уведомлении реализована кнопка **«⏳ Отложить на 5 минут»**,  который автоматически переносит таймер.

### 🚀 Как запустить Итоговый проект (Практика 4-5)

Проект состоит из двух частей: Сервер (Backend) и Клиент (Frontend). Их нужно запускать параллельно в разных терминалах.

#### 1. Запуск Сервера (Backend + Swagger)

Сервер отвечает за базу данных товаров и автоматическую документацию API (Swagger).

```bash
cd Practice 1
npm install
node app.js
```

#### 2. Запуск Клиента (Frontend)
```bash
cd client
npm install
npm start
```

#### 3. Запуск PWA
```bash
cd practice_pwa
node server.js
```
